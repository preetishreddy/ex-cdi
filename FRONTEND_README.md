## Getting Started (Run the project locally)

**Step 1 — Activate the virtual environment**

On Windows (PowerShell):
```
.\venv\Scripts\Activate.ps1
```

On Mac/Linux:
```
source venv/bin/activate
```

You will see `(venv)` appear in your terminal. That means it worked.

**Step 2 — Install dependencies**
```
pip install -r requirements.txt
```

**Step 3 — Set up your environment variables**

Copy `.env.example` and create your own `.env` file:
```
cp .env.example .env
```

Then open `.env` and fill in the values. Ask the backend team for the correct credentials.

**Step 4 — Run the server**
```
python manage.py runserver
```

Open your browser and go to `http://127.0.0.1:8000`

## Environment Variables — `.env`

All secrets and environment-specific values are stored in a `.env` file in the
project root. This file is **never committed to git** — each developer has their own.

Use `.env.example` as a template. It has all the keys with empty values:

```
SECRET_KEY=          ← Django's security key, get this from the backend team
DEBUG=               ← True for local development, False on live server

DB_NAME=             ← name of the PostgreSQL database
DB_USER=             ← database username
DB_PASSWORD=         ← database password
DB_HOST=             ← usually localhost
DB_PORT=             ← usually 5432
```

**Rule: Never paste real credentials into `.env.example`. Keep it empty.**

## File Guide for the Front-End Team

---

### `.env` — Your Local Secrets

This is the only file where you set credentials. You get the values from the
backend team. Never share this file or commit it to git.

---

### `config/settings.py` — Project Configuration

The main settings file. Sensitive values like `SECRET_KEY` and database
credentials are no longer hardcoded here — they are read from `.env` automatically.

Only touch these sections if needed:

**ALLOWED_HOSTS** — If you are running the backend on a custom IP or domain, add it here.
```python
ALLOWED_HOSTS = ['localhost', '127.0.0.1']
```

**STATIC_URL** — URL prefix for all static files (CSS, JS, images).
Do not change this unless the backend team agrees.
```python
STATIC_URL = 'static/'
```

**TEMPLATES → DIRS** — If you want Django to look for HTML files in a
top-level `templates/` folder, add the path here.
```python
'DIRS': [BASE_DIR / 'templates'],
```

**LANGUAGE_CODE and TIME_ZONE** — Change these if the project needs a
different language or timezone.
```python
LANGUAGE_CODE = 'en-us'
TIME_ZONE = 'UTC'
```

---

### `config/urls.py` — URL Routing

Controls what happens when someone visits a URL. Check this file to see
what pages and API endpoints exist.

Current URLs:
| URL | What it does |
|-----|-------------|
| `/admin/` | Django admin panel (backend use only) |

New pages added by the backend team will appear here.

---

### `my_app/views.py` — Page Logic

Every page on the website is a function in this file. When the backend team
adds a new page, it starts here. You do not edit this file but you can read
it to understand what data each page is working with.

---

### `my_app/models.py` — Database Structure

Defines what data is stored and how it is structured. Read this to understand
what fields exist (e.g. title, content, created_at). You do not edit this file.

---

### `my_app/templates/` — HTML Files (front-end lives here)

This folder does not exist yet — the backend team will create it.
Once created, this is where your HTML files go.

```
my_app/
└── templates/
    └── my_app/
        └── home.html    ← your HTML goes here
```

---

### `my_app/static/` — CSS, JS, Images (front-end lives here)

This folder does not exist yet — the backend team will create it.
Once created, this is where your CSS, JavaScript, and image files go.

```
my_app/
└── static/
    └── my_app/
        ├── css/
        ├── js/
        └── images/
```

## Files You Must NOT Touch

| File | Why |
|------|-----|
| `manage.py` | Django's internal command tool. Changing it breaks everything. |
| `.env.example` | Template file — only add new keys, never put real values in it. |
| `config/__init__.py` | Empty file Python needs internally. Do not touch. |
| `config/wsgi.py` | Only used when deploying to a live server. |
| `config/asgi.py` | Only used when deploying to a live server. |
| `config/settings.py` → `SECRET_KEY` | Now reads from `.env`. Do not hardcode a value here. |
| `config/settings.py` → `DEBUG` | Now reads from `.env`. Do not hardcode a value here. |
| `config/settings.py` → `MIDDLEWARE` | Security layers. Do not touch. |
| `config/settings.py` → `DATABASES` | Now reads from `.env`. Do not touch. |
| `config/settings.py` → `INSTALLED_APPS` | List of active apps. Backend team only. |
| `my_app/admin.py` | Backend use only. |
| `my_app/apps.py` | App config. Backend team only. |
| `my_app/migrations/` | Auto-generated database history. Never edit manually. |
| `my_app/tests.py` | Backend use only. |
| `venv/` | Virtual environment. Never edit anything inside here. |
